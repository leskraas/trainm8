Integration Hub — three UI directions

Where connecting activity sources should live, and what pasting an Intervals.icu API key feels like. All three mocks use the app's real tokens (teal primary, 0.625rem radius, card/border/muted values) and real copy — including the honest states: syncing, importing history, needs re-authorization, and coming soon.

Recommendation: Variant A — the settings hub.

Connection management is settings-shaped: visited rarely, acted on deliberately. A stacked card list keeps every provider's state readable at a glance, gives the API-key flow room to explain itself inline, and each future provider is one more card. Variant B's dialog flow is worth stealing for mobile; Variant C shows why management shouldn't stay on the inbox.

Variant A · recommended

Settings hub — stacked provider cards

A dedicated /settings/integrations page. Connected sources first with live state, then available ones with their connect flow expanded inline, then honest coming-soon cards. The phone view shows the same page reflowed.

Wins
  • Every state visible at once — no clicks to discover a broken connection.
  • Room to be honest — backfill progress, dual-source warning, and the partner-program story fit without crowding.
  • Cheap to extend — a new provider is one more card (ADR 0014's copy-the-folder promise, in UI form).

Wins
  • Zero navigation — source state sits where activities land.
  • The chip strip is a good pattern regardless: A and B keep it as the inbox's summary line.
Costs
  • The inbox stops being an inbox — a work queue visited after every workout shouldn't carry connect flows, re-auth errors, and partner-program stories.
  • Chips can't hold the API-key flow; it would end up in a dialog anyway.
  • Coming-soon providers have no natural home here at all.

Decisions shared by every variant

  • Plain-language states, one vocabulary: Connected · Importing history · Needs re-authorization · Coming soon. Operational sync state comes from the job queue, never a status column (CONTEXT.md).
  • Honest latency copy for Intervals.icu: no webhook exists for personal API keys, so the card says "checked daily — sync now to pull the latest immediately" instead of pretending it's live.
  • The dual-source warning appears whenever Strava and Intervals.icu are both active — duplicates stay athlete-driven, and we say so.
  • Coming soon means the truth: the Garmin/Suunto cards name the partner-program gate and point to Intervals.icu as today's route for that data. No waitlists.
  • Provider identity tiles are placeholders — monogram tiles in brand-adjacent colors; real marks are a build-time asset pass.

Tokens mirror app/styles/tailwind.css (light + dark). Type is a system stand-in for Public Sans — the artifact sandbox can't fetch webfonts. Design record: ADR 0026 · feature issue #199 · branch claude/integration-interface-redesign-k6odd3.
